## 1. Календарь — отображение записей

**Текущая проблема:**
- Записи не влезают в ячейки календаря
- Слишком длинный формат названия

**Было:**
```
[1234] Тестовый проект - Протокол: 2026-01-26 12-50-46.mp4
```

**Решение — компактный формат:**
```
[КОД] Название проекта
```

**Примеры:**
```
[1234] Тестовый проект
[3105] ЦОД Доброград
```

- Цвет события по статусу (🔴 critical, 🟡 attention, 🟢 stable)
- Детали (дата файла, тип) — в модалке при клике

## 3. Роли на проекте

### Иерархия ролей (снизу вверх)

```
superuser
    ↓ наследует
  admin
    ↓ наследует
 manager
    ↓ НЕ наследует (отдельная ветка)
  viewer
    ↓ НЕ наследует (отдельная ветка)
   user
```

### Описание ролей

| Роль | Описание | Ключевые права |
|------|----------|----------------|
| **user** | Рядовой сотрудник | Загрузка файлов, `/history`, БЕЗ риск-брифа |
| **viewer** | Наблюдатель | Доступ в `/dashboard`, но ЧТО видит? _(уточнить)_ |
| **manager** | Руководитель проектов | Риск-бриф + все совещания по СВОИМ проектам |
| **admin** | Администратор | Права manager + раздаёт права пользователям |
| **superuser** | Супер-админ | Права admin + назначает админов |

### Матрица доступа

| Возможность | user | viewer | manager | admin |
|-------------|------|--------|---------|-------|
| Загрузка файлов | ✅ | ✅ | ✅ | ✅ |
| `/history` | Свои | Все по своим проектам | Все по своим проектам | Все |
| `/dashboard` | ❌ | ✅ упрощённый | ✅ полный | ✅ полный |
| Риск-бриф в модалке | ❌ | ❌ | ✅ | ✅ |
| Проекты | По назначению | По назначению | По назначению | Все |
| Админ-панель | ❌ | ❌ | ❌ | ✅ |
| Назначение ролей | ❌ | ❌ | ❌ | ✅ |
| Назначение админов | ❌ | ❌ | ❌ | Только superuser |

### Решение: Viewer = Руководитель проекта(ов)

**Кто это:** Руководитель/администратор проекта (может быть несколько проектов)

**Ключевые особенности:**
- Загружает файлы ✅
- Доступ в `/dashboard` ✅
- Видит историю ВСЕХ user'ов по своим проектам ✅
- Может иметь доступ к нескольким проектам ✅
- Риск-бриф ❌ (не показываем)

**Что видит в дашборде:**
- Календарь с протоколами по своим проектам
- **Упрощённая модалка** при клике:
  - **Экспертный анализ** — общая оценка ситуации
  - **Краткое содержание** — о чём было совещание
  - **Кнопки скачивания:** transcript, tasks (БЕЗ риск-брифа)
  - БЕЗ рисков, БЕЗ "Требует внимания"

## 4. Модалка — структура и редактирование

### Источники данных (актуальные)

| JSON в БД | Содержит |
|-----------|----------|
| `basic_report_json` | meeting_summary, expert_analysis, tasks[] |
| `risk_brief_json` | overall_status, executive_summary, atmosphere, concerns[], risks[] |

~~`ReportAnalytics`~~ — deprecated, не используется

---

### Приоритеты отображения в модалке

**Риск-бриф:**

| Приоритет | Поле | Как показывать |
|-----------|------|----------------|
| 🔴 | overall_status | Бейдж в заголовке |
| 🔴 | executive_summary | Всегда видно |
| 🔴 | concerns[] | Секция "Открытые вопросы" |
| 🟡 | atmosphere | Компактный бейдж |
| 🟡 | risks[] (title, severity, decision) | Список с аккордеонами |
| 🟢 | risks[] (drivers, evidence) | Внутри аккордеона |
| 🟢 | abbreviations | Tooltip или скрыто |

**Таски:**

| Приоритет | Поле | Как показывать |
|-----------|------|----------------|
| 🔴 | description | Всегда |
| 🔴 | responsible | Всегда |
| 🟡 | deadline | Всегда |
| 🟡 | priority | Цвет/иконка |
| 🟢 | evidence, time_codes | В аккордеоне |

---

### Порядок секций в модалке (manager)

```
1. Заголовок + статус (overall_status)
2. Краткое содержание (meeting_summary)
3. Экспертный анализ (expert_analysis)
4. Атмосфера (atmosphere)
5. Открытые вопросы (concerns[])
6. Риски (risks[]) — с аккордеонами
7. Задачи (tasks[]) — компактный список
8. Скачивания: transcript.docx, tasks.xlsx, risk_brief.pdf
```

---

### Редактирование артефактов

**Кто может:** только manager
**История версий:** не нужна
**Транскрипт:** read-only

**Что редактируется:**

| Артефакт | Поля | Результат |
|----------|------|-----------|
| Риски | title, decision, responsible, severity | → regenerate risk_brief.pdf |
| Concerns | добавить/удалить/изменить | → regenerate risk_brief.pdf |
| Tasks | description, responsible, deadline | → regenerate tasks.xlsx |

**Флоу:**
```
[✏️ Редактировать] → Форма → Сохранить
    ↓
PATCH /api/reports/{id}/risk_brief  (или /tasks)
    ↓
Обновить JSON в БД
    ↓
Перегенерировать PDF/XLSX
    ↓
UI обновляется
```

---

### Модалка viewer (упрощённая)

```
1. Заголовок + статус
2. Краткое содержание
3. Экспертный анализ
4. Открытые вопросы (concerns[])  ← добавили
5. Задачи (tasks[])
6. Скачивания: transcript.docx, tasks.xlsx (БЕЗ risk_brief.pdf)
```

**Viewer видит:** summary, expert_analysis, concerns, tasks
**Viewer НЕ видит:** risks, атмосферу, risk_brief.pdf
**Viewer НЕ может:** редактировать
