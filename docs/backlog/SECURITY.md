# Бэклог: Безопасность

## 1. JWT-токен в sessionStorage → HttpOnly Cookie

**Приоритет:** Средний (технический долг)
**Версия:** 2.x
**Статус:** Отложено

### Проблема
JWT-токен хранится в `sessionStorage`, что делает его доступным через JavaScript.
При XSS-атаке злоумышленник может извлечь токен и использовать для авторизации.

### Текущее смягчение
- Внутрикорпоративное приложение без выхода в открытый интернет
- Нет пользовательского контента (комментарии, чаты), через который можно внедрить скрипт
- CSP-заголовки ограничивают выполнение inline-скриптов

### Что нужно для фикса
1. **Backend:** Отправка JWT через `Set-Cookie` с флагами `HttpOnly`, `Secure`, `SameSite=Strict`
2. **Backend:** Настройка CORS `allow_credentials=True`
3. **Backend:** Защита от CSRF (double-submit cookie или синхронизирующий токен)
4. **Frontend:** Убрать хранение токена в Zustand store
5. **Frontend:** Убрать Authorization-заголовок из Axios-интерсепторов (куки отправляются автоматически)
6. **Frontend:** Добавить `withCredentials: true` в Axios

### Затронутые файлы
- `backend/core/auth/` — логика выдачи и проверки токенов
- `frontend/src/stores/` — Zustand auth store
- `frontend/src/api/` — Axios-интерсепторы
- SSO-интеграция с Hub

## 2. Unsafe torch.load — защита кэша моделей

**Приоритет:** Низкий
**Статус:** Частично решено

### Проблема
`torch.load()` с `weights_only=False` позволяет выполнить произвольный код
при загрузке вредоносного файла модели (pickle deserialization attack).

### Текущее смягчение
- Фильтр `is_trusted_model` проверяет источник моделей
- Dockerfile: `chmod -R 555` на кэш моделей базового образа
- Prod compose: volume `models` монтируется как `:ro` на API-контейнере
- Worker-GPU контейнер использует именованный Docker volume (не host mount)

### Что можно дополнительно
- После первоначального скачивания моделей переключить volume на `:ro` и для worker-gpu
  (требует ручного шага при первом деплое или init-контейнера)
- Рассмотреть `torch.load(..., weights_only=True)` для совместимых моделей
- Добавить checksum-верификацию скачанных моделей

## 3. Дополнительные улучшения безопасности (низкий приоритет)

- **Rate limiting на auth-эндпоинтах** — защита от brute-force (сейчас есть общий rate limit)
- **Audit log** — логирование действий админов (создание/удаление пользователей)
- **Secrets rotation** — процедура ротации GEMINI_API_KEY, JWT_SECRET, DB passwords
